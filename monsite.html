INTRODUCTION GENERALE

Au tout dédut, nous nous sommes intéressés à la formation du Devops, c'est à dire à quoi ça sert? comment ça fonctionne et en quoi cela pourrait nous être utile. Alors nous avons commencer pour nous intéresser au fonctionnement des réseaux informatiques, la gestion et l'administration des systêmes informatiques, les langages de codage informatique et la gestion des projets informatiques en passant par les depots locaux et dépots distants. Dans la suite de cette page nous vous montrerons étape après étape, et en détail ce que nous avons appris jusqu'ici.

DIFFERENTS MODULES ABORDES

Jusqu'ici, voici les différents modules de formation que nous avons abordés:

  1. Comprendre et maîtriser la conception et le fonctionnement des réseaux informatiques.

    2. Le streaming, les mails, le cloud ...etc, ont un point en commun: L'interconnectivité. dans ce cours nous nous nois sommes familiarisés avec l'outil de simulation notamment Packettracer. Pour en savoir plus sur ce module, bien vouloir visiter la page openclassrooms

  2. Administration de système Linux

    3. Ici il est question de mettre en place Linux sur notre ordinateur avec une distribution comme Ubuntu, interagir avec notre ordinateur en entrant des lignes de commande dans un terminal et d'utiliser les bons outils pour exploiter Linux dans notre vie professionnelle. Ce cours est accessible à tous, donc Rendez-vous sur le site openclassrooms

  3. Code de base en langage python

    4. Python est un langage de programmation assez généraliste, c’est-à-dire qu’il est à peu près possible de tout faire avec : des sites et applications web, des applications mobiles, des scripts personnels, des applications de bureau, de l’analyse de données et même des jeux vidéo ! Concernant la data, Python est le langage le plus utilisé, en particulier pour les traitements et manipulations de données, la data visualisation et le développement d’algorithmes d’intelligence artificielle. Pour en savoir plus visiter la page suivante openclassrooms

  4. Gestion des projets informatiques à travers Git/Github

    5. Imaginez, vous avez passé une semaine à écrire votre code, et donc produit votre première version, la V1. Votre collègue vous a fait des suggestions pour l’améliorer. Vous allez donc créer une deuxième version de votre code, la V2. Pour garder un historique de votre travail et éviter les mauvaises surprises (bugs, perte de votre travail), vous aurez besoin d’un gestionnaire de versions. Donc de Git pour gerer les dépots en local sur votre ordinateur et Github pour gérer ces codes à distance dans le cloud. Le cours est accessible à tous à travers le lien openclassrooms
